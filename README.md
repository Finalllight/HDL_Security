# HDL_Security

HDL_Security 研究  
主要问题：当前 HDL 语言在硬件设计上存在多种安全问题

由于技术节点的持续进步导致芯片制造成本的增加，设计公司开始采用全球化供应链。在这种所谓的无厂商业模式中，设计公司将制造、测试和封装外包给第三方设施。然而，外包给潜在的不受信任实体导致了不同的安全威胁。这些威胁可能在不同供应链阶段或由最终用户实施。

## 一. 硬件设计的安全重要性说明

1. 硬件设计是一个复杂的流程，可能会出现重复性、易出错和效率低下的问题。
2. 硬件支撑对软件系统极其重要，且硬件作为最底层，漏洞可能导致整个系统安全失效。硬件漏洞，尤其是寄存器传输级 RTL 阶段漏洞，往往比软件漏洞更具破坏性。
3. 芯片制造后无法像软件一样通过补丁修复，因此提前发现问题极其重要。

## 二. 当前安全验证存在的不足

1. 主流验证技术（测试、形式化验证、模糊测试等）需黄金参考模型或断言，但断言生成需手动操作和专业知识，易出错且扩展性差。
2. 安全断言需检测“不应该存在的弱点”，与功能断言逻辑不同，难以通过自动挖掘生成，现有自动化断言生成方法不适用于安全场景。

## 三. 面临的安全问题

1. 硬件木马：硬件木马是破坏设备完整性和功能的恶意隐蔽电路修改，它是硬件安全中最显著的安全挑战之一，被报道为最隐蔽的网络攻击形式之一。
2. 侧信道攻击：攻击者利用电路泄漏，如功耗和电磁辐射，从系统中提取敏感信息，基于功耗和基于缓存的攻击最为突出。
3. 逆向工程：此类攻击允许攻击者从集成电路中提取专有信息，分析 IC 以揭示其设计和功能。
4. SoC 安全：现代计算系统基于片上系统（SoC）构建，因为它们通过使用多个知识产权（IP）核提供了高度集成，一个 IP 核中的漏洞可能会影响整个系统的安全性。
5. 故障注入

## 四. 对应解决方案

核心思路：基于机器学习和大模型去缓解以上安全问题。LLM 的最新进展为自动化数字硬件设计提供了巨大潜力，可以被用于 RTL 漏洞检测和缓解、评估框架、比较分析、案例研究以及未来对专用架构、改进数据集和 EDA 工具集成上。

1. 利用 LLMs 来生成、植入和检测硬件木马，并实现漏洞修复。
2. 侧信道攻击：利用 LLMs 和基于图的技巧来应对侧信道漏洞，自动化漏洞检测，生成保护措施，提高安全分析的易解释性，从而实现了早期阶段的漏洞识别。
3. 逻辑混淆
4. 利用自然语言处理和 LLM 来提取或生成安全属性、识别漏洞并生成全面的安全策略。
